## Strategy cheatsheet from prior cases in this area

Below is a cheatsheet of ATOMIC STRATEGY BULLETS curated from prior
cases on structurally similar problems. Each bullet captures a
strategy, workflow step, tool quirk, convention, or pitfall worth
knowing for this kind of case. Bullets are accompanied by their usage
counters: `helpful` is how often the bullet was cited on a case that
was judged correct, `harmful` is how often it was cited on an
incorrect case, `usage` is the total cite count.

Use this as procedural memory, not as evidence and not as an answer
key. First solve the current case from its actual prompt, files, and
tool results. Consult a bullet only when its trigger matches the
current task structure, then adapt the method to the current data.

Do NOT copy specific numerical values, file paths, named entities, or
named references from a bullet — values in bullets are illustrative;
the current case has different actual ones. If a bullet conflicts with
the current prompt, tool output, or file contents, the current case
wins. Use bullets to choose checks, workflows, conventions, and
failure guards; verify the result independently.

{bullets_block}

When you submit your `final_answer` tool call, in the `reasoning`
field, end with a separate line listing the `bullet_id` of every
bullet that actually influenced your work. Do not cite bullets you only
skimmed or ignored. Use this exact format:

    <citations>[bullet-1, bullet-7]</citations>

If you consulted no bullets, write:

    <citations>[]</citations>

The citations line is internal bookkeeping; it is stripped from your
reasoning before the deliverable is reviewed.
