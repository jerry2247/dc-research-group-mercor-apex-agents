## Reusable Code and Tool-Call Snippets
<memory_item>
<description>
To extract specific text from a PDF (such as statute sections or legal provisions), first call pdf_server_pdf with action="search" using the key term or citation as query (case-insensitive), then follow with action="read_pages" on the returned page number(s). Use max_results=5 and context_chars=100 for search.
</description>
<example>
pdf_server_pdf({"request":{"action":"search","file_path":"/Reorganization/State Statutes/<STATE>/<pdfname>.pdf","query":"194A.713","case_sensitive":false,"max_results":5,"context_chars":100}})
</example>
</memory_item>
<memory_item>
<description>
To read multi-page .docx agreements, call docs_server_docs with action="read_content" and page_index (starting at 0) to retrieve paragraphs sequentially; repeat for additional pages as needed. Paths must start with / and end with .docx.
</description>
<example>
docs_server_docs({"request":{"action":"read_content","file_path":"/Magnolia Documents/Contracts/<agreement>.docx","page_index":0}})
</example>
</memory_item>
<memory_item>
<description>
To create a new .docx document (e.g., a legal memorandum or analysis), call docs_server_docs with action="create", specifying directory (folder path) and file_name (ending with .docx) separately, along with content as an array of valid content blocks (each object must have a "type" field such as "heading" or "paragraph" with required fields populated; empty objects or incorrect parameter names like file_path will fail validation).
</description>
<example>
docs_server_docs({"request":{"action":"create","directory":"/Litigation Strategy/Wrongful Death Documents/Tom Buchanan","file_name":"Memo.docx","content":[{"type":"heading","text":"Title","level":1},{"type":"paragraph","text":"Body text here."}]}})
</example>
</memory_item>
<memory_item>
<description>
To compute dates, day of week, or time intervals in the sandbox when direct Python execution fails, use code_execution_server_code_exec with a shell date command (e.g., date -d 'YYYY-MM-DD' +%A or similar formatting).
</description>
<example>
code_execution_server_code_exec({"request":{"code":"date -d '2025-08-06' +%A"}})
</example>
</memory_item>

## Solution Strategies for Recurring Task Shapes
<memory_item>
<description>
For compliance reviews of assisted living residency agreements against state statutes, first locate and extract the exact statutory requirement(s) from the state PDF, then read the target agreement document(s) page-by-page, and compare each enumerated statutory element against the agreement language for presence or omission. When multiple similar agreement files exist, select the most general or state-matched version rather than a state-specific variant.
</description>
<example>
Locate statute PDF → search/read target section → read agreement → map each statutory item (e.g., notice periods, fee provisions) to agreement text.
</example>
</memory_item>
<memory_item>
<description>
For evaluating a motion for summary judgment on a wrongful death claim, locate the complaint and moving party's supporting affidavit, then review opposing evidence such as a deposition to determine whether it raises a genuine issue of material fact on key elements (duty, breach, causation). If factual disputes exist on those elements, the motion is likely denied.
</description>
<example>
Locate complaint and affidavit in litigation folder → read deposition of key witness (e.g., on care protocols) → compare evidence for triable issues under civil procedure rules.
</example>
</memory_item>
<memory_item>
<description>
For compliance reviews against specific regulatory sections (e.g., NY Part 487 on meals/snacks, TX Chapter 276 on deposits, KY 194A.713 on notice, or CA CCR Title 22 §87507), locate the relevant PDF in /Reorganization/State Statutes/<STATE>/ or root-level rules PDFs, use pdf_server_pdf search for key terms (multiple targeted searches if needed), read_pages on matches, and compare provided operational facts against the exact rule language.
</description>
<example>
Locate state PDF → search for regulation keywords (e.g., "meals", "snacks", "pet deposit", "service animal", "87507") → read relevant pages → check if facts satisfy the rule.
</example>
</memory_item>

## Formulas, Definitions, and Conventions
<memory_item>
<description>
KRS 194A.713(7) and analogous state provisions require a minimum 30-day notice for fee structure changes in assisted living lease agreements; absence of any such notice language constitutes non-compliance.
</description>
<example>
Statutory element (7): "Minimum thirty (30) day notice provision for a change in the community's fee structure."
</example>
</memory_item>
<memory_item>
<description>
KRS 194A.713(14) requires that an assisted living lease agreement include grievance policies that minimally address issues related to confidentiality of complaints and the process for resolving grievances between the resident and the assisted living community; absence of such policies constitutes non-compliance.
</description>
<example>
Statutory element (14): "Grievance policies that minimally address issues related to confidentiality of complaints and the process for resolving grievances between the resident and the assisted living community."
</example>
</memory_item>

## Pitfalls, Edge Cases, and Verification Checks
<memory_item>
<description>
When multiple residency agreement files exist, verify the selected file matches the task scope (general template vs. state-specific or resident-specific version) before comparison; California RCFE forms may contain 60-day notice language that satisfies a 30-day minimum but is irrelevant to other states.
</description>
<example>
Confirm agreement file name and content scope before mapping to statute (e.g., avoid using CA-specific form for KY compliance check).
</summary>
</memory_item>
<memory_item>
<description>
When creating new .docx files with docs_server_docs action="create", the content parameter must be a list of valid block objects (each with a "type" field such as "heading" or "paragraph"); passing empty objects, empty lists, or structures missing the type tag causes repeated validation errors. Always specify directory and file_name separately.
</description>
<example>
Use content=[{"type":"heading","text":"Title","level":1},{"type":"paragraph","text":"Body text."}] ; do not pass {} or invalid tags.
</summary>
</memory_item>
<memory_item>
<description>
For tasks involving identification of objectionable testimony in deposition transcripts under a cited evidence rule (such as NY Rule of Evidence 10.03), locate and read the deposition file (e.g., Nurse Deposition.docx) using sequential read_content calls to access the specific paragraph referenced in the task, then analyze statements for violations like lack of personal knowledge, speculation, or hearsay.
</description>
<example>
Use docs_server_docs to read Nurse Deposition.docx pages until paragraph 8 is located; flag statements in that paragraph that fail the rule's requirements for admissibility.
</summary>
</memory_item>
<memory_item>
<description>
When checking compliance with a specific regulation (e.g., NY Part 487 on meal planning or similar), confirm the exact statutory text and requirements match the scenario facts (such as advance planning interval or snack frequency) before concluding compliance or non-compliance; search multiple related terms if initial results are narrow.
</description>
<example>
Confirm the regulatory text on "meals" or "snacks" directly addresses the provided operational detail (e.g., planning two and a half weeks in advance) before using it.
</summary>
</memory_item>
